# Лабораторная работа №3. Коллекция задач: итераторы и генераторы

## Описание работы
Третий этап разработки платформы посвящен созданию эффективной системы управления очередью задач `TaskQueue`. Основной фокус сделан на реализации протоколов итерирования и использовании ленивых вычислений. 

В данной работе реализовано разделение ответственности между контейнером данных и механизмом обхода (итератором), что позволяет интегрировать коллекцию в стандартные конструкции Python (циклы, преобразования типов) и обеспечивать высокую производительность при фильтрации больших объемов данных.

## Функционал
* **Собственный итератор**: Реализация класса `TaskQueueIterator` с поддержкой протокола `__next__` и корректной обработкой исключения `StopIteration`.
* **Ленивая фильтрация**: Использование генераторов (`yield`) в методах фильтрации по статусу и приоритету, что исключает избыточное копирование данных в памяти.
* **Магические методы коллекции**: Поддержка `__len__` для определения размера очереди, `__iter__` для встраивания в циклы и `__add__` для удобного добавления новых задач.
* **Архитектурная связность**: Полная совместимость с моделью `Task` из Лабораторной работы №2, включая доступ к данным через дескрипторы в процессе итерации.
